05 — Emergency Personnel Dashboard (Health & Security — Separate Access)
Separate access path.
Emergency personnel do NOT use public sign-up. Access provisioned by city/org admin via dedicated credentials. Health team and security team are separate audiences.

  • Active alerts live feed
  • Send broadcast alert
  • Health info directory
  • Contact directory

  • User name (if available)
  • Time alert was sent
  • Live location pin (tap → map)
  • Status: New / Acknowledged / Resolved
Health alerts → health team only. Security alerts → security team only.

  • Compose message

  • Confirm & send

  • Search by name / user ID
  • Blood type, allergies, health type
  • Emergency contact details
Read-only. Accessible only after credentials verified.
Registered Users
  • Total registered count
  • User list (name, contact, area)
  • Filter by location / status
Full alert routing: User taps Emergency (Step 1 — live location captured) → selects Health or Security (Step 2) → confirms (Step 3) → alert + location auto-sent. Health alert → pushed to Health team dashboard with push notification. Personnel opens alert → views health profile (blood type, allergies) + live map → calls user → dispatches ambulance. Security alert → pushed to Security team dashboard. Officer calls user. If no answer → uses live location to dispatch physically.

Role split logic: Sign Up → User vs Service Provider (1 prompt). SP → Vendor vs Transport (2nd prompt). Only relevant fields shown per role. Lean onboarding.
Emergency 3-step design: Location captured at Step 1 (before confirmation) — maximises response time even if user abandons the flow. 3 taps prevent accidental triggers.
Map screens: User sees nearest transport providers before confirming a request. Transport sees both their location and user's pickup simultaneously. Mirrors Uber/Bolt UX.
Vendor onboarding: Step 1 required (basic info). Step 2 optional (storefront). Vendors go live immediately after Step 1. Storefront completed later via Settings.
Future scope (v2+):   Wallet payment logic & integration · Vendor payout system · Transport background verification flow · Admin panel for city/org credential provisioning · Push notification infrastructure · Multi-language support · Accessibility (screen reader, font scaling) · Rating moderation tools · City operator analytics dashboard
